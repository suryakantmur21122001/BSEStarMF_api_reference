# Authentication API Reference (BSE StAR MF)

## Overview

This folder contains the reference documentation and Python structure for the **Authentication API (`getPassword`)** used in the BSE StAR Mutual Fund platform.

The Authentication API is the first step required before calling any other API services such as order entry, SIP registration, redemption, or order status.

This API establishes a **secure session between the client system (company backend)** and the **BSE StAR MF server**.

---

## API Information

**API Name:** Authentication API
**Method:** `getPassword`
**Endpoint:**
https://bsestarmfdemo.bseindia.com/MFOrderEntry/MFOrder.svc/Secure

**Protocol Used:** SOAP (XML-based Web Service)

---

## Purpose of the API

The purpose of the `getPassword` API is to **authenticate the member system** with the BSE StAR MF web service.

Once authentication is successful, the server generates an **Encrypted Password (Session ID)** which is required for all further API requests.

This session ID acts as a **temporary login token** between the company backend system and the BSE server.

Session validity:

* 1 hour for MFOrder service
* 5 minutes for other services

---

## Authentication Request Parameters

| Parameter | Description                                               |
| --------- | --------------------------------------------------------- |
| UserId    | Login ID provided by BSE                                  |
| Password  | Password assigned to the user                             |
| PassKey   | Random alphanumeric string generated by the client system |

The PassKey is used to increase security and session entropy.

---

## Process Flow (End-to-End)

### Step 1 — User Initiates Request

A client (investor or distributor) performs an action on the company application.

Example actions:

* Purchase Mutual Fund
* Start SIP
* Redeem Mutual Fund

The company backend must first authenticate with the BSE StAR MF server.

---

### Step 2 — Backend Generates SOAP Request

The backend system prepares a SOAP XML request containing:

* User ID
* Password
* PassKey

This request follows the structure defined in the API documentation.

---

### Step 3 — API Request Sent to BSE Server

The SOAP request is sent to the secure API endpoint:

`https://bsestarmfdemo.bseindia.com/MFOrderEntry/MFOrder.svc/Secure`

The request is transmitted using HTTPS for secure communication.

---

### Step 4 — Server Validates Credentials

The BSE StAR MF server verifies:

* User ID
* Password
* Member status
* Login validity

If credentials are correct, the server generates an **Encrypted Password (Session ID).**

---

### Step 5 — Server Sends Authentication Response

The server returns a SOAP response containing:

Response Code | Encrypted Password

Example response:

100|UWQ/yLAS7KmiHyExtsKKq3DuHTWuyGOzy0P4s38PSL8nA==

Where:

* **100** → Login Successful
* **Encrypted Password** → Session ID used for future API requests

---

### Step 6 — Backend Stores Session ID

The company backend system stores the **Encrypted Password (Session ID)**.

This session ID is then used for calling other APIs such as:

* Order Entry API
* SIP Registration API
* Redemption API
* Order Status API
