# Lumpsum Purchase & Redemption Order Entry API Reference

This folder contains the **reference documentation and Python implementation structure** for the **BSE Star MF Lumpsum Purchase & Redemption Order Entry API**.

The Lumpsum Order Entry API allows member institutions and distributors to place **mutual fund purchase and redemption orders** through the BSE Star MF platform using web services.

---

## Purpose of the API

The main purpose of this API is to allow distributors and financial institutions to **automatically place lumpsum mutual fund transactions** from their backend systems.

Using this API, the system can:

* Submit mutual fund **purchase orders**
* Submit **redemption orders**
* Send transaction data securely to BSE servers
* Receive order confirmation responses

---

## Transaction Types

The API supports two types of transactions:

### Purchase (P)

A purchase transaction is used when an investor wants to **buy mutual fund units**.

### Redemption (R)

A redemption transaction is used when an investor wants to **sell mutual fund units** from an existing folio.

---

## Order Entry Method

The web service method used for placing orders is:

```
orderEntryParam
```

This method accepts multiple parameters including transaction details, client details, scheme information, and authentication credentials.

---

## Order Entry Process Flow

The process of placing an order using this API works as follows:

1. The **client or distributor system** collects investor and transaction details.
2. The backend system prepares a **SOAP request message** using the required parameters.
3. The request is sent to the **BSE Star MF server**.
4. The server validates the transaction and processes the order.
5. BSE generates a **unique order ID** for the transaction.
6. The server sends a **response message** back to the distributor system confirming the transaction status.

---

## Important Transaction Fields

Some key parameters used in the order entry request include:

* `TransCode` → Transaction type (NEW/MOD/CXL)
* `TransNo` → Unique transaction reference number
* `ClientCode` → Investor client code
* `SchemeCd` → Mutual fund scheme code
* `BuySell` → Purchase or Redemption indicator
* `OrderVal` → Transaction amount
* `Qty` → Redemption quantity
* `FolioNo` → Investor folio number
* `KYCStatus` → Client KYC verification status

## Notes

* Each transaction must have a **unique transaction number (TransNo)**.
* The **Order ID is generated by the BSE server** after successful processing.
* Authentication credentials such as encrypted password and pass key are required to send requests.
* This API works in integration with the **BSE Star MF Web Services platform**.
